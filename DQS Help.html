Welcome

Welcome to Database Quick Search. DQS, created by Evan Elrod, is a tool used to rapidly find drawings, spreadsheets, PDF files, and anything else saved on the various network drives. A separate program builds a database of all files on a semi-daily basis. DQS, in turn, scans the database for file information (which is much faster than searching the actual drives) and takes you to the file's location.

The purpose of creating this program was so that we wouldn’t have to wait around for AutoCAD, Windows, etc. to find a file. As the amount of files on our server continues to balloon, it is my hope that this program will save you a great deal of time and allow you to be more efficient in your tasks.

Getting Started

Although intended to be intuitive, there may be some features of the program which may be better utilized after understanding how they may benefit you.
Let's start with basic usage:
   Step 0: (Optional) Check/un-check the checkboxes applicable to the file you're looking for. Upon first-use, you may want to save your settings if you wish to use these parameters on future searches. See the "Features & Usage" section for more information.
   Step 1: Begin typing the name or number of the file you're searching for.
   Step 2a: To open the file, click the "Open file" button (or you can press the [Enter]-key or use the [Alt]+[O] key-combination).
   Step 2b: To see the file in Windows Explorer, click the "Show In Explorer" button (or you can use the [Alt]+[S] key-combination).
If your search yielded multiple results, you'll be presented with another window displaying the results. From here you can double-click the desired file. See "Features & Usage", item "8", for more information.

If you are unsure of a feature, briefly rest your mouse over it for a small tooltip which will provide a summarized explanation. For a full explanation into all aspects of the program, please look through the next section, "Features & Usage".

1. Menu Bar

The menu-bar is divided into four sections: "File", "Presets", "History, and "Help".

"File" - Contains standard, common, or general commands.



  • "Local Database" - Lists local drives and when they were built.
    • [Individual Local Drives] - Each local drive is shown and when it was last updated. Clicking it will update the individual drive.
    • "Build All" - Updates all local drives if more than one is listed. Not shown if only one local drive is available.
  • "Settings" - Contains the various customizable options below:
    • "Colors" - Set the color-scheme base on your preferences. Click here to see all color schemes.
    • "Default Button" - From the options below, set the button that corresponds with the [Enter]-key.
      • "Open File" - Opens the file in its default program.
      • "Show In Explorer" - Takes you to the file's location in file explorer.
    • "Default Search Parameters" - Check-boxes checked/un-checked when you start the program. Options below:
      • "Custom Saved" - What you've personally saved using [Ctrl]+[S].
      • "Previous Use" - Restores the settings of your very last use. This is set by default.
"Presets" - Contains predefined 'packages' of checked/un-checked check boxes, suited to specific departments/needs.

  • "1.) Bronx Engineering" - Specifically suited for finding files in the Engineering Department. Hotkey: [Ctrl]+[1]
  • "2.) Abbey Engineering" - Specifically suited for finding files related to archive Abbey engineering. Hotkey: [Ctrl]+[2]
  • "3.) Bronx Aftermarket" - Specifically suited for finding files in the Aftermarket Department. Hotkey: [Ctrl]+[3]
  • "9.) Maximum Search" - This will do a complete search for all files indexed on the databases that you have access to. Hotkey: [Ctrl]+[9]
  • "0.) Custom" - In case you've changed your check-boxes, this will revert to your saved settings. Hotkey: [Ctrl]+[0]
  • "My Local Drive(s) Search" - This will only search your local drives. Hotkey: [Ctrl]+[M]
  • "Previous Search Parameters" - This will use the same parameters as your most previous search. Hotkey: [Ctrl]+[P]
  • "Save Custom Search Settings" - This save the checkboxes and other settings. Hotkey: [Ctrl]+[S]
"History" - Options to manage and view previous searches.

  • "Show History" - This will bring up a list of previous searches with the following options:

    • "Search File Again" - This will re-search the filename you selected, also allowing you to search with different parameters.
    • "Open File" - This will open the file you selected (without searching) in the applicable program, based on the file's type.
    • "Show In Explorer" - This will take you to the selected file's location (without searching) in file explorer.
  • "Clear History" - As implied, this will remove all entries from your search history.
"Help" - Go here if you encounter problems with or need information about the "Database Quick Search" program.

  • "View Help" - Open this help page that you are currently viewing.
  • "What's New" - Lists the detailed view of all changes and updates to the program.
  • "User Statistics" - Lists all users who have used the program within the last 30 days and details about their installations and searches.
  • "About" - Shows basic program version information and lists when the databases were last updated.

2. File Type and Network Path Selection



Clicking the "Edit Search Criteria" button or either list box will open the "Edit Search Criteria" dialog box. In this box you can add filetypes and local/network paths. Once your selection is complete, press enter or click the "Update Search Criteria" button. The list boxes will update based on your selections.

Filetypes are determined by the extension of the filename and determine what programs will open them by default.
The file-types to choose one or more of:
  • ".dwg" - The standard drawing extension, usually opened with AutoCAD, TrueView, etc.
  • ".dwf" - An Autodesk Design Web Format file. Usually opened with Autodesk Design Review.
  • ".pdf" - Many documents can be saved in this format, usually opened with an Adobe product, such as Adobe Reader.
  • ".tif" - A standard picture format. Scanned drawings are usually saved with this extension. Can be opened with many programs such as Microsoft Picture Viewer.
  • ".xls(x)" - Common extensions for Microsoft Excel. Almost always opens with Microsoft Excel.
  • "Other" - This covers all other extensions besides those shown above. Use cautiously as this will make your search take a bit longer.
  • "Select All/None" - Use this to quickly check/un-check all check-boxes in this selection set.
The drives and paths are located on the network server and are where we store the bulk of our files.
The drives to choose one or more of:
  • "Aftermarket (R)" - Aftermarket drawings and projects.
  • "Company (N)" - Company-related files.
  • "Engineering (Q)" - Engineering files.
    • "Abbey Drawings" - Archived Abbey projects.
    • "Abbey Projects" - Abbey projects.
    • "Bronx Drawings" - Engineering files, drawings, and models.
    • "WIP" - Engineering work-in-process files, largely unused.
  • "NAS (Z)\Scanned Drawings" - Scans of physical drawings.
    • "Abbey Scanned Drawings" - Scans of Abbey physical files.
    • "Bronx Scanned Drawings" - Scans of Bronx physical files.
  • "Procurement (P)" - POs, SAP-related files, etc.
  • "Projects (T)" - Manuals, project management, transmittals, etc.
  • "Public (Y)" - Catalogs, public documents and software, misc. folders, etc.
  • "Sales (S)" - Sales and presentation documents pertaining to past and current jobs.
  • "Select All/None" - Use this to quickly check/un-check all check-boxes in this selection set.
Local drives are detected and shown as options to choose. Most will have one or two drives. If a local database hasn't been made or is older than seven days, you'll be prompted to update. The update process is brief.

4. Miscellaneous

Various options to modify your search results and how it's opened.
The miscellaneous options are:

  • "Suppress results in *\old*, *commercial*, etc." - If checked, results which have the following in their file-path will be filtered out: "\old", "\ref", "\received", "\recieved", "void", "deleted", "original", and "zbronx". If any additions or subtractions need to be made from this list, it can be easily modified and is located here: Y:\Utilities & Software\Database Quick Search\Statistics\Filter List.txt.
  • "Open file as 'READ-ONLY' (if applicable)" - If checked and applicable, your result will be opened in read-only mode.
  • "Abbey Search" and button - If checked, it inserts the wildcard asterisk (*) between each typed character. Enclose any portion that you know to be consecutive in single quotes ('). Alternatively, use the "Abbey Helper" button to format the text appropriately. Note that clicking the button auto-checks the "Abbey Search" checkbox, as it is necessary for the program to understand how to search.
    Examples:
    8k4-300 would find B-08K004-00300-00 and 018K14-00230-00 (+2,000 others)
    8k4-'300' would find B-08K004-00300-00 and C-08K043-00300-00 (+20 others)
    8k'4-''300' would find B-08K004-00300-00

5. File-Name Must Start With....

Check this check-box if you know your desired file(s) begins with what you type in the text-box. Uncheck this checkbox if your desired file(s) might not begin with what you type in the textbox.

Examples:
Most Bronx drawing files start with a known 5 or 6-digit number, so you would check the "Must Start With...." checkbox to make your searches faster.
If you're searching for all cylinder .pdf catalogs using the search-text "cyl", you would likely want to uncheck the "Must Start With...." checkbox, as it is likely that there are many files that you may want, but do not start with "cyl".
If you're searching for an Abbey drawing using the search-text "8K4-123", you would want to uncheck the "Must Start With...." checkbox, as there may be files that begin "X-08K004", etc. Note that checking the "Abbey Search" checkbox nullifies the "Must Start With...." checkbox.

Note that having this checkbox unchecked may significantly increase the time it takes to find your file. It is recommended that this box remains checked unless you strongly suspect that your file doesn't begin with what you've typed in the search textbox.

8. User Action Buttons

After typing for what to search, you can press the [Enter]-key, which is defaulted to the "Open File" button. This default can be changed by going to "File" » "Settings" » "Default Button". The [Alt]+[O] key-combination does this same task. The [Alt]+[S] key-combination is the same as clicking the "Show In Explorer" button.

  • "Open File" - Clicking this button will open the file in your computer's default program to open such files. Hotkey: [Alt] + [O]
  • "Show In Explorer" - Clicking this button will take you to the file's location in Windows Explorer. Hotkey: [Alt] + [S]

If multiple files are found, a new window will pop-up, displaying a complete list of all matches.

It is sorted by filename by default, but feel free to sort as needed to help you find what you're looking for. From that point, you can double-click (or select and press [Enter]) the file you want (also, see options below). Or, if you want to keep the results window open, you can right-click individual files to take you to the file's location in file explorer.
If you had previously clicked the "Open File" button, you will be able to open multiple files (using [shift], [ctrl], or a crossing-window). Though, you may only select one file if you had previously clicked the "Show In Explorer" button. This is because it would open up a separate explorer window for each file, and that is probably undesirable.

Two additional features have been added to this window:

  • "Export list to .csv" button - The entire list will be exported to a CSV-file (can open with Excel) which you can save wherever you choose. Its formatting is similar to that of the "Multiple Files Found...." window.
  • Right-click an item » "Open in Explorer (without closing window)" - As implied, you can open the file's location in file explorer while keeping the "Multiple Files Found...." window open. You can open as many as you'd like, but just one at a time. If multiple files are selected when you right-click, only the one that was right-clicked or had primary focus will be shown in file explorer.

Questions & Answers

Q: Why are the checkboxes not staying how I had them?
A: After you've checked/unchecked whichever checkboxes suit your needs, save your settings by going to "Presets" in the menu-bar and selecting "Save Settings" (which can also be done with the key-combination: [ctrl]+[s]).
Alternatively, you can set the default parameters to be as what you had last searched by going to "File" » "Settings" » "Default Search Parameters" and selecting "Previous Use".

Q: Why is [Windows-Key] + [F] not working anymore?
A: This is likely an issue with the operating system or group policy settings. Regardless, to restore functionality, in file explorer go to %appdata%\Microsoft\Windows\Start Menu\Programs\Startup and double-click "DQSWinF.exe".

Q: Why is the program not finding my file (I know it exists)?
A: Make sure the checkboxes you have checked make sense for the file you're looking for and try searching again.
If the file was added to the server recently, after the database was last built, the file won't be in the database until the next update. You can check when the databases were last built by going to "Help" in the file menu and clicking "About".
If the file has been renamed, moved, or deleted, you'll get a warning notifying you of that and will have the option of opening file explorer to the file's last known location.
If it still cannot find the file, please let me know as it could be an error in the program.

Q: Why are my searches taking so long?
A: The most likely reason is that you have all, or too many, checkboxes checked for searching drives and filetypes. Try narrowing your search parameters by unchecking drives or filetypes that you're not searching for. Refer to "Features & Usage" for more information.
Another possible reason is that you're not directly connected to the network (that is, if you're connected over the internet instead of the LAN). In order to load the database to memory it must be downloaded, so it is dependent on the transfer speed of both your and Fives Bronx's internet connection.
An additional possibility is that your computer is lacking sufficient RAM or processing power.

Q: Why is the program finding files I didn't search for?
A: Much has been done to make searches easy and results expected. However, not everything works as intended, so please do let me know if you detect an aberrant result.

Q: When and how often are the databases updated?
A: The databases are updated five days per week, twice per day, Monday through Friday, at 6:15 a.m. and 12:15 p.m. Obviously, if the server is down for maintenance or other reasons, the databases will not be updated that day. You can find out how up-to-date the databases are by going to "Help" in the menubar, and selecting "About". If needed, the databases can be updated manually.
